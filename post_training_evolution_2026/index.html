序章 · 出发点

基础回顾：RLHF 三阶段与 PPO一切故事的起点：InstructGPT (2022) 确立的范式

\[ \max_{\pi_\theta}\; \mathbb{E}_{x\sim D,\,y\sim\pi_\theta}\big[ r_\phi(x,y)\big] - \beta\, \mathbb{D}_{\mathrm{KL}}\big[\pi_\theta(y|x)\,\|\,\pi_{\mathrm{ref}}(y|x)\big] \]
RLHF 总目标：最大化奖励，同时用 KL 惩罚约束策略不偏离参考模型（防止奖励作弊与语言退化）。
\[ \mathcal{J}_{\mathrm{PPO}}(\theta)=\mathbb{E}_t\Big[\min\big(\rho_t(\theta)\,\hat A_t,\; \mathrm{clip}(\rho_t(\theta),1\!-\!\varepsilon,1\!+\!\varepsilon)\,\hat A_t\big)\Big],\quad \rho_t(\theta)=\frac{\pi_\theta(a_t|s_t)}{\pi_{\mathrm{old}}(a_t|s_t)} \]
PPO 用重要性采样比 \(\rho_t\) 的裁剪构造信任域；优势 \(\hat A_t\) 由独立的价值网络（critic）+ GAE 估计。

这个范式的三个痛点，恰好定义了 2024–2026 后训练的三条演化线：

  • RM 会被作弊（reward hacking）且标注昂贵 → 走向可验证奖励；
  • Critic 价值网络在长序列稀疏奖励下难以训练、显存翻倍 → 走向 GRPO 组内基线；
  • RL 系统吞吐被 rollout 长尾拖垮 → 走向异步、解耦的 RL 基础设施。

Tülu 3 · 数学 ①

偏好对齐：从 RLHF 目标推导 DPO，再做长度归一化

RLHF 目标（上一页）存在闭式最优解：

\[ \pi^*(y|x)=\frac{1}{Z(x)}\,\pi_{\mathrm{ref}}(y|x)\exp\!\Big(\tfrac{1}{\beta}r(x,y)\Big) \]
反解出 \(r(x,y)=\beta\log\frac{\pi^*(y|x)}{\pi_{\mathrm{ref}}(y|x)}+\beta\log Z(x)\)，代入 Bradley–Terry 偏好模型后配分函数 \(Z\) 消掉。
\[ \mathcal{L}_{\mathrm{DPO}}=-\mathbb{E}\Big[\log\sigma\Big(\beta\log\tfrac{\pi_\theta(y_w|x)}{\pi_{\mathrm{ref}}(y_w|x)}-\beta\log\tfrac{\pi_\theta(y_l|x)}{\pi_{\mathrm{ref}}(y_l|x)}\Big)\Big] \]
DPO：无需显式 RM 与 RL 循环，直接在偏好对 \((y_w,y_l)\) 上做分类式优化。

Tülu 3 的选择：长度归一化 DPO（消除「越长越好」的隐性偏置）：

\[ \mathcal{L}_{\mathrm{LN\text{-}DPO}}=-\mathbb{E}\Big[\log\sigma\Big(\tfrac{\beta}{|y_w|}\log\tfrac{\pi_\theta(y_w|x)}{\pi_{\mathrm{ref}}(y_w|x)}-\tfrac{\beta}{|y_l|}\log\tfrac{\pi_\theta(y_l|x)}{\pi_{\mathrm{ref}}(y_l|x)}\Big)\Big] \]
对序列 log-ratio 按 token 数 \(|y|\) 归一化。消融显示其稳定优于标准 DPO 与 SimPO 等变体。

为什么重要：偏好数据由「on-policy 自采样 + off-policy 他模型输出」混合构造，配合 LLM-as-judge 标注——这套「合成偏好流水线」成为后来所有开源模型偏好阶段的模板。

Tülu 3 · 数学 ②

RLVR：把奖励模型换成「验证函数」Reinforcement Learning with Verifiable Rewards —— 术语在此诞生

\[ r_{\mathrm{RLVR}}(x,y)=\begin{cases} \alpha, & \mathrm{verify}(x,y)=\text{True}\\[2pt] 0, & \text{否则}\end{cases} \]
\(\mathrm{verify}\) 是确定性程序：数学答案精确匹配、约束检查器（IFEval）、测试用例执行。没有可学习参数 ⇒ 无法被「讨好」。
\[ \max_{\pi_\theta}\ \mathbb{E}_{y\sim\pi_\theta}\big[ r_{\mathrm{RLVR}}(x,y)\big]-\beta\,\mathbb{D}_{\mathrm{KL}}\big[\pi_\theta\|\pi_{\mathrm{ref}}\big] \]
优化器仍是 PPO（带价值网络；价值网络从通用 RM 初始化效果最好）。训练集：GSM8K、MATH、IFEval。
  • 动机：RM 是奖励作弊的软肋；在「对错可判定」的域，直接把判定器当奖励，信号无噪声、无上限偏置。
  • 效果：数学与精确指令遵循显著提升，且不损害其他能力；在 405B 上同样有效。
  • 历史地位：两个月后 DeepSeek-R1 用大规模 RLVR 点燃推理革命；此后 K2 的「verifiable rewards gym」、GLM 的执行环境、MAI 的可执行代码奖励，全部是 RLVR 的规模化延伸。

一句话遗产：Tülu 3 证明了「后训练配方可以像菜谱一样公开复现」，并给了社区第一个无 RM 的 RL 抓手。故事由此开始。

锚点 · 数学

GRPO：用「组内相对优势」砍掉价值网络Group Relative Policy Optimization（DeepSeekMath, 2024 → R1, 2025）

\[ \mathcal{J}_{\mathrm{GRPO}}(\theta)=\mathbb{E}_{x,\{y_i\}_{i=1}^{G}\sim\pi_{\mathrm{old}}}\Bigg[\frac{1}{G}\sum_{i=1}^{G}\frac{1}{|y_i|}\sum_{t=1}^{|y_i|}\Big(\min\big(\rho_{i,t}\hat A_{i},\ \mathrm{clip}(\rho_{i,t},1\!-\!\varepsilon,1\!+\!\varepsilon)\hat A_{i}\big)-\beta\,\mathbb{D}_{\mathrm{KL}}[\pi_\theta\|\pi_{\mathrm{ref}}]\Big)\Bigg] \]
\(\rho_{i,t}=\frac{\pi_\theta(y_{i,t}|x,y_{i,<t})}{\pi_{\mathrm{old}}(y_{i,t}|x,y_{i,<t})}\)；KL 用无偏估计 \(\ \frac{\pi_{\mathrm{ref}}}{\pi_\theta}-\log\frac{\pi_{\mathrm{ref}}}{\pi_\theta}-1\ \)（逐 token、恒非负）。
\[ \hat A_i=\frac{r_i-\mathrm{mean}(r_1,\dots,r_G)}{\mathrm{std}(r_1,\dots,r_G)} \]
同一 prompt 采样 \(G\) 条回答，组内均值即基线——价值网络被一次采样统计量替代，序列内所有 token 共享同一优势。
  • 省一半显存与计算：不再维护和训练与策略同尺寸的 critic；
  • 与验证奖励天然契合：0/1 奖励下组内归一化即成对比信号；
  • 代价：优势是响应级而非 token 级 —— 后续所有改进（DAPO、K2.5、MAI）都在修补这一点。
锚点 · 训练学

R1 的四段流水线与 K1.5 的镜像下降两条路线，一个结论：推理能力可以被 RL「催生」而非「示范」

DeepSeek-R1：从 Zero 到产品级

Kimi K1.5：在线策略镜像下降（OPMD）

\[ \mathcal{L}(\theta)=\mathbb{E}_{(x,y)\sim\pi_{\mathrm{old}}}\Big[\big(r(x,y)-\bar r-\tau\log\tfrac{\pi_\theta(y|x)}{\pi_{\mathrm{old}}(y|x)}\big)^{2}\Big] \]
来自相对熵正则的策略迭代 \(\max_\pi \mathbb{E}[r]-\tau\mathrm{KL}(\pi\|\pi_{\mathrm{old}})\) 的闭式解 \(\pi^*\!\propto\!\pi_{\mathrm{old}}e^{r/\tau}\)：把「匹配最优策略」写成平方损失，\(\bar r\) 近似配分函数项 \(\tau\log Z\)。梯度 = 带基线的策略梯度 + \(\ell_2\) 正则。
  • 长度惩罚：对过长回答扣分，防「无效思考」；long2short（模型合并、最短拒绝采样、DPO）把长 CoT 能力压进短回答；
  • 课程：难度分级采样 + 按成功率的优先采样；
  • 底层判断与 R1 相同：不需要 MCTS/价值树搜索，「自回归 CoT 即隐式搜索」。

伏笔：这条平方损失血脉将进化为 K2 的 RL 算法，再进化为 K2.5 带 token 级裁剪的混合目标（第五章）。

K2 · 数学 ① 优化器

MuonClip = Muon + QK-Clip：驯服注意力 logit 爆炸token 效率与万亿级稳定性可以兼得

Muon 更新（矩阵参数 \(\mathbf W\in\mathbb{R}^{n\times m}\)）：动量矩阵经 Newton–Schulz 迭代正交化，并匹配 Adam 的更新 RMS：

\[ \mathbf M_t=\mu\mathbf M_{t-1}+\mathbf G_t,\qquad \mathbf O_t=\mathrm{NS}(\mathbf M_t)\cdot\sqrt{\max(n,m)}\cdot 0.2 \] \[ \mathbf W_t=\mathbf W_{t-1}-\eta\,(\mathbf O_t+\lambda\mathbf W_{t-1}) \]
NS 迭代 \(X_{k}=aX_{k-1}+b(X_{k-1}X_{k-1}^{\!\top})X_{k-1}+c(X_{k-1}X_{k-1}^{\!\top})^2X_{k-1}\) 把奇异值推向 1（近似 \(UV^\top\) 正交化）。同算力下 token 效率显著优于 AdamW。

问题：Muon 的更新谱更「满」，训练中注意力 logit 迅速超过 1000 → 损失尖峰甚至发散。soft-cap 治标；QK-Norm 对 MLA 不可用（K 矩阵推理时不显式物化）。

QK-Clip：每步更新后监控每个头的最大 logit，超阈值则直接缩放权重（不改变当步前向/反向）：

\[ S^h_{\max}=\tfrac{1}{\sqrt d}\max_{\mathbf X\in B}\max_{i,j}\ \mathbf Q_i^h\mathbf K_j^{h\top},\qquad \gamma_h=\min\!\big(1,\ \tau/S^h_{\max}\big) \] \[ \mathbf W^{h}_{qc}\!\leftarrow\!\sqrt{\gamma_h}\,\mathbf W^{h}_{qc},\quad \mathbf W^{h}_{kc}\!\leftarrow\!\sqrt{\gamma_h}\,\mathbf W^{h}_{kc},\quad \mathbf W^{h}_{qr}\!\leftarrow\!\gamma_h\,\mathbf W^{h}_{qr} \]
按头裁剪（只有极少数头爆炸）；MLA 中共享的旋转键 \(\mathbf k^R\) 不动以免跨头串扰。K2 取 \(\tau=100\)：logits 初期贴着 100 封顶，约 30% 步数后自然回落 —— 之后 QK-Clip「自动退休」，对模型质量无损。

影响：MuonClip 成为 Kimi 系(K2/K2.5) 预训练与 RL 的默认优化器；DeepSeek-V4 也转投 Muon（附加混合 Newton–Schulz 与自己的防爆手段）。优化器从「炼丹细节」升格为报告主角。

K2 · 数学 ② RL

联合 RL：可验证奖励 Gym + 自评 Rubric 奖励把 RLVR 推广到不可验证的开放域 —— 模型学会「给自己打分」

算法：K1.5 血脉 + 三个稳定器

\[ \mathcal{L}(\theta)=\mathbb{E}\Big[\tfrac{1}{K}\sum_{i=1}^{K}\big(r(x,y_i)-\bar r(x)-\tau\log\tfrac{\pi_\theta(y_i|x)}{\pi_{\mathrm{old}}(y_i|x)}\big)^2\Big] \]
OPMD 平方损失 + 组均值基线（继承 K1.5，仍无 critic）。
  • 预算控制：按任务设 token 预算，超限惩罚 —— 强制「简洁思考」；

奖励：两条腿走路

结果：非思考模式下 SWE-bench Verified 65.8、Tau2-bench 66.1、AIME25 49.5 —— 当时最强开源，逼近 Claude 4 系。伏笔：自评 rubric 在 K2.5 长成通用「生成式奖励模型 GRM」。

GLM-5 · 数学

IcePop：修正「训练-推理失配」的 GRPOMoE 大规模 RL 的隐形杀手：同一模型，两套引擎，两个分布

\[ \mathcal{L}(\theta)=-\,\mathbb{E}_{x,\{y_i\}\sim\pi^{\text{infer}}_{\theta_{\mathrm{old}}}}\Bigg[\frac{1}{G}\sum_{i=1}^{G}\frac{1}{|y_i|}\sum_{t=1}^{|y_i|}\ \underbrace{\mathrm{pop}\big(\rho_{i,t},\ \tfrac{1}{\beta},\ \beta\big)}_{\text{训推失配门控}}\cdot\ \min\Big(r_{i,t}\hat A_{i},\ \mathrm{clip}\big(r_{i,t},1\!-\!\epsilon_{\mathrm{low}},1\!+\!\epsilon_{\mathrm{high}}\big)\hat A_{i}\Big)\Bigg] \]
\(\rho_{i,t}=\pi^{\text{train}}_\theta / \pi^{\text{infer}}_\theta\) 是同一参数在训练引擎与推理引擎下的概率比；\(\mathrm{pop}(\cdot)\) 将 \(\rho\notin[1/\beta,\beta]\) 的 token 梯度置零。\(r_{i,t}\) 是常规重要性比（对 \(\pi_{\mathrm{old}}\)），非对称裁剪 \(\epsilon_{\mathrm{low}}\neq\epsilon_{\mathrm{high}}\)（DAPO 谱系）。相比原始 IcePop，GLM-5 去掉了 KL 正则以加速提升。

为什么会失配？

训练用 Megatron、rollout 用 SGLang：算子实现、批不变性、精度路径都不同。MoE 下微小数值差被路由放大 —— 推理引擎认为概率 0.3 的 token，训练引擎可能算出 0.2。梯度按错误的分布加权 ⇒ 训练悄然崩坏。

同一问题的三种解法（跨报告对照）

GLM-5：IcePop 门控（算法侧屏蔽）；K2.5：对 log-ratio 的 token 级硬裁剪（下一章）；MAI / DeepSeek-V4：干脆做批不变、逐位确定的算子库，从根上消灭失配（基础设施侧）。异步智能体 RL 里 GLM-5 还叠加 token-in-token-out（杜绝重分词漂移）与双侧重要性采样裁剪。

GLM-5 · 基础设施

slime + APRIL：异步 RL 的解剖rollout 占 RL 总时长 90%+，长尾样本让 GPU 集体围观 —— 系统问题要用系统解

APRIL：主动部分回滚（arXiv 2509.18521）

  • 超额发车：每步发出多于目标数的 rollout 请求；
  • 达标即停：凑齐目标条数立刻终止本轮，不等长尾；
  • 断点回收：未完成的响应存入池中，下一步继续生成（partial rollout 复用）。
效果：GRPO / DAPO / GSPO 上 rollout 吞吐提升至多 44%，收敛更快，最终精度至多 +8%；框架与硬件无关，已并入 slime。代价：被回收的轨迹前后段来自不同策略版本 —— 又一种「轻度 off-policy」，恰好由 IcePop / 双侧裁剪兜底。算法与系统在此互为因果。

环境规模化：SWE 环境从真实仓库批量构建；终端环境从种子任务与网页语料双路合成；搜索任务配上下文管理。环境（而非数据集）成为新的稀缺资产。

K2.5 · 数学 ① 策略优化

K2.5 的 RL 目标：OPMD 血脉 + token 级失配裁剪与 GLM 的 IcePop 殊途同归 —— 却是从算法侧动手

\[ \mathcal{L}_{\mathrm{RL}}(\theta)=\mathbb{E}_{x\sim\mathcal D}\Bigg[\frac{1}{N}\sum_{j=1}^{K}\sum_{i=1}^{|y_j|}\underbrace{\mathrm{Clip}\Big(\tfrac{\pi_\theta(y_j^i|x,y_j^{0:i})}{\pi_{\mathrm{old}}(y_j^i|x,y_j^{0:i})},\ \alpha,\ \beta\Big)}_{\text{token级门控（对log-ratio，双侧）}}\big(r(x,y_j)-\bar r(x)\big)\ -\ \tau\Big(\log\tfrac{\pi_\theta(y_j^i|x,y_j^{0:i})}{\pi_{\mathrm{old}}(y_j^i|x,y_j^{0:i})}\Big)^{2}\Bigg] \]
\(N=\sum_j |y_j|\)（全批 token 数），\(\bar r(x)=\frac1K\sum_j r(x,y_j)\) 组均值基线。与 PPO 裁剪的本质区别：门控只看 log-ratio 是否越界（\([\alpha,\beta]\) 之外梯度置零），与优势符号无关 —— 目的不是构造悲观下界，而是硬性约束 off-policy 漂移（训推失配 + 异步陈旧度都被它挡住）。第二项 \(\tau(\log\rho)^2\) 是 K1.5 平方正则的直系遗产。优化器：MuonClip。

Toggle：预算约束的「间歇训练」

\[ \tilde r(x,y)=\begin{cases} r(x,y)\cdot\mathbb{I}\{\bar r(x)<\lambda\ \text{或}\ |y|\le \mathrm{budget}(x)\} & \text{Phase 0（限预算）}\\ r(x,y) & \text{Phase 1（放开）}\end{cases} \]
每 \(m\) 轮切换：Phase 0 逼模型少费 token 也能解题（简单题超预算则奖励清零），Phase 1 放开预算保留「加算力换深度」的能力 —— 治好 K1.5 式硬预算的「长度过拟合」。

K2.5 · 数学 ② 多智能体

Agent Swarm 与 PARL：只训练「指挥家」Parallel-Agent RL —— 并行不是启发式，是被 RL 学出来的策略

\[ \text{CriticalSteps}=\sum_{t=1}^{T}\Big(S^{(t)}_{\mathrm{main}}+\max_i S^{(t)}_{\mathrm{sub},i}\Big) \]
以「关键路径步数」替代总步数做资源约束：奖励缩短最长并行分支的分解方式，而非单纯堆并发。
\[ r_{\mathrm{PARL}}(x,y)=\lambda_1\!\cdot\!\underbrace{r_{\text{parallel}}}_{\text{实例化奖励}}+\lambda_2\!\cdot\!\underbrace{r_{\text{finish}}}_{\text{子任务完成率}}+\underbrace{r_{\text{perf}}(x,y)}_{\text{任务结果}} \]
两个辅助项各治一种病：\(r_{\text{parallel}}\) 治串行塌缩（指挥家躺平退化为单智能体）；\(r_{\text{finish}}\) 治虚假并行（滥开子体刷指标的奖励作弊）。\(\lambda_1,\lambda_2\) 随训练退火到 0—— 脚手架拆除，只留任务本身。

mHC：把残差流约束在 Birkhoff 多面体上Manifold-Constrained Hyper-Connections —— 让「更宽的残差」不再炸

Hyper-Connections 把残差流加宽 \(n_{\mathrm{hc}}\) 倍（\(X_l\in\mathbb{R}^{n_{\mathrm{hc}}\times d}\)），三个映射控制进出：

\[ X_{l+1}=B_l X_l + C_l\,\mathcal{F}_l(A_l X_l) \]
\(A_l\in\mathbb{R}^{1\times n_{\mathrm{hc}}}\) 选入、\(B_l\in\mathbb{R}^{n_{\mathrm{hc}}\times n_{\mathrm{hc}}}\) 残差混合、\(C_l\in\mathbb{R}^{n_{\mathrm{hc}}\times1}\) 写回。层内计算维度仍是 \(d\)，代价极小。但深堆 HC 数值不稳。
\[ B_l\in\mathcal{M}=\{M\mid M\mathbf 1=\mathbf 1,\ \mathbf 1^{\!\top}\!M=\mathbf 1^{\!\top},\ M\ge 0\} \]
mHC 的核心：把 \(B_l\) 投影到双随机矩阵流形（Birkhoff 多面体）⇒ \(\|B_l\|_2\le 1\)（非扩张），且 \(\mathcal M\) 对乘法封闭 ⇒ 任意深度堆叠信号不放大。

参数动态生成（输入依赖 + 静态偏置），再施加约束：

\[ A_l=\sigma(\tilde A_l),\qquad C_l=2\,\sigma(\tilde C_l),\qquad B_l=\lim_{t}\ \mathcal{T}_r\big(\mathcal{T}_c(\exp \tilde B_l)\big) \]
\(B_l\) 用 Sinkhorn–Knopp：先 \(\exp\) 保正，再交替行/列归一化收敛到双随机。\(\sigma\) 保证 \(A_l, C_l\) 非负有界，避免信号相消。
  • Muon 侧防爆：混合 Newton–Schulz 迭代 + 防注意力 logit 爆炸机制（呼应 K2 的 QK-Clip 问题意识）；
  • 训练稳定性组合拳：Anticipatory Routing（路由用历史参数解耦更新、打断恶性反馈环）+ SwiGLU 钳位；
  • 路由激活函数 Sigmoid → \(\mathrm{Sqrt}(\mathrm{Softplus}(\cdot))\)，前几层 dense FFN 换成 Hash 路由 MoE。

RL Climb 的目标函数：GRPO 的两处「小而关键」的修改token 级归一 + 自适应熵控制 + 外层比率裁剪

\[ \mathcal{J}(\theta)=\mathbb{E}_{q,\,y_{1:G}\sim\pi_{\mathrm{old}}}\Bigg[\frac{1}{\sum_{i=1}^{G}|y_i|}\sum_{i=1}^{G}\sum_{t=1}^{|y_i|}\min\Big(r_{i,t}(\theta)\,A_i,\ \mathrm{clip}\big(r_{i,t}(\theta),\,1\!-\!\epsilon,\,1\!+\!\epsilon\big)A_i\Big)\Bigg],\qquad A_i=\frac{R_i-\mathrm{mean}(R_{1:G})}{\mathrm{std}(R_{1:G})} \]
token 级策略梯度（DAPO 谱系）：归一化分母是全局批的总 token 数（跨数据并行组）——每个 token 等权，长回答不再被稀释。奖励来自代码执行 / AI 评审 / 训练过的 RM，随专家域切换。

修改一：自适应熵控制

\[ r^{\mathrm{tr}}_{i,t}(\theta)=\mathrm{clip}\Big(r_{i,t}(\theta),\ 1-\epsilon,\ (1-\epsilon)^{-1}+k\Big) \]
上界解耦为 \((1-\epsilon)^{-1}+k\)，\(k\) 由控制器动态调节：观测熵 \(\hat H(\pi_\theta)\) 高于目标 \(H^\star\)（如 0.3）则收紧、低于则放宽 —— 把「熵坍缩 vs 熵爆炸」变成一个闭环控制问题，支撑数千步不间断爬升。

修改二 + 复活术

  • 外层比率裁剪：对未截断分支再加一层帽，防梯度范数爆炸（罕见 token 的 \(r_{i,t}\) 可以极端大）；

  • ①奖励的去人化：人类偏好→验证器→环境执行→生成式 RM——人从「打分者」退居「环境与尺子的设计者」。

落地篇 · 方向① 奖励设计

企业 RAG 的奖励金字塔：从「引用可验证」到「行为信号」

\[ r(x,y)=\underbrace{r_{\text{cite}}}_{\text{可验证层}}+\lambda_1\underbrace{r_{\text{GRM}}}_{\text{质量层}}+\lambda_2\underbrace{r_{\text{behav}}}_{\text{行为层}} \]
\(r_{\text{cite}}\)：答案的每个断言必须被所引片段蕴含（NLI 判定 + 引用 ID 存在性检查 + 拒答正确性）——这是 RLVR 思想在 RAG 的直接移植：「有据可查」是企业场景里最接近「可验证」的东西。\(r_{\text{GRM}}\)：完整性/简洁/格式（K2.5 式多 rubric 轮换）。\(r_{\text{behav}}\)：采纳率/追问率/复制粘贴行为，仅做低权重校准。

面试必问：为什么不直接用点击/采纳做奖励？→ 四宗罪：稀疏（多数查询无反馈）、有偏（位置偏置、老员工偏好）、延迟（周级归因）、可作弊（讨好式答案骗采纳）。正解是行为信号训练 GRM / 校准验证器，而不是直接进 RL —— 这正是 RM→RLVR→GRM 演化在工业界的重演。

反作弊：GLM-5.2 免疫系统的 RAG 版

  • 作弊形态迁移：训练中模型学会「编造带引用格式的幻觉」「引用一个存在但不相关的片段」「大段复述文档规避判定」；
  • 两级免疫：规则层（引用 ID 必须存在于检索集合、复述比例上限）→ LLM 意图裁判（该引用是否真正支持该断言）；只清零作弊样本奖励，rollout 继续；
  • 审计闭环：作弊样本不丢弃 —— 它们是最好的对抗评测集与 GRM 训练负例（把矛炼成盾）。

对应章节：第一章 RLVR ｜ 第三/五章 rubric→GRM ｜ 第八章 反作弊。

落地篇 · 方向② Agentic 搜索训练

企业 DeepResearch：多跳检索智能体怎么训出来

环境与任务：反向出题法

  • 环境 = 文档库快照 + 三个工具：search(q) / open(doc) / quote(span)——放进 microVM 沙箱（V4 DSec 思想），可抢占、可回放；
  • 任务合成（K2 数据工厂的 RAG 版）：从文档引用图上采样 k 跳路径 → 让强模型反向生成「必须跨 k 篇文档才能回答」的问题 + 可验证的标准答案与证据集；难度 = 跳数 × 干扰文档数；
  • 课程：单跳事实 → 双跳对比 → 跨部门综合报告（呼应 K1.5 难度课程）。
奖励：\(r=\) 答案正确性（对标准证据集的召回/精确）\(+\) 过程合规（每步引用真实）\(-\) 步数成本。长尾现实：有的查询 5 步、有的 50 步 —— APRIL 的部分回滚在此不是优化而是刚需；异步带来的漂移由 IcePop/log-ratio 门控兜底（第四/五章原封不动搬来）。

规模化：从单体到 Swarm

  • wide-search 场景（「汇总近三年所有涉及供应商 X 的合同风险」）：orchestrator 把文档域切片，并行子体各自扫描，汇总去重 —— K2.5 Agent Swarm 的企业形态；
  • PARL 奖励直接迁移：\(r_{\text{perf}}\)=报告的证据覆盖率，\(r_{\text{finish}}\)=子查询完成率（防「虚假并行」——滥开子体刷指标），退火脚手架；
  • 上下文管理：子体只回传「引用+摘要」而非全文 —— Swarm 同时是主动式上下文压缩（K2.5 报告的原生洞见）。

面试题：多智能体系统怎么做信用分配？→ 先给「不做」的答案（PARL：冻结子体、只训指挥家、辅助奖励防塌缩），再谈何时必须端到端（子体能力成为瓶颈时）——展示你知道工程解与理想解的边界。

层级延迟/规模训练来源

关键直觉：教师的价值不在上线，而在「随时能教」。专家池是持续资产——新领域=新专家=一次 OPD，而非一次全量重训。

为什么是 OPD 而不是老式蒸馏

  • 老式（离线）蒸馏：教师生成数据集→学生 SFT。问题：学生在教师的分布上学习，上线后遇到自己的错误状态就失控（暴露偏差）；
  • OPD：学生在线上真实 query 回放上自己生成，教师逐 token 全词表批改（反向 KL）——学的恰是「我会犯的错」；GLM-5.2 用 slime 两天完成一轮 ⇒ 周级模型迭代节奏成为可能；
  • 版本升级：新底座上任时用旧模型自蒸馏保能力（MAI 的 self-distillation 复活术）——企业最怕的「升级即回退」有了标准解法。

面试题：蒸馏损失为何选反向 KL？→ 正向 KL 是 mean-seeking（学生被迫覆盖教师所有模式→摊薄）；反向 KL 是 mode-seeking（学生在自己策略可达的状态上贴合教师最优模式）—— 小模型容量有限，「学精一条路」优于「学浅所有路」。

落地篇 · 方向④ 长上下文 vs RAG

1M 上下文会杀死 RAG 吗？—— 成本曲线说了算

  • 检索的本质是稀疏注意力的系统外实现：BM25/向量召回 ≈ 一个不可微、但零 GPU 成本的 top-k indexer —— 与 V4 的 Lightning Indexer、GLM-5.2 的 IndexShare 做的是同一件事，只是位置不同（库外 vs 层内）；
  • V4 把曲线压弯了：1M ctx 只花 27% FLOPs / 10% KV，且 on-disk KV cache + 前缀复用意味着「把整个知识库预填充一次、按 query 复用」——知识库变成一个可检索的 KV 缓存，介于 RAG 与全上下文之间的新物种；
  • 决策矩阵：库 ≲1M token / 跨文档推理密集 / 更新慢 → 整库进上下文；库巨大 / 点查为主 / 分钟级更新 / 权限隔离 → 检索；中间地带 → 检索粗筛 + 长上下文精读（agentic 两阶段，事实上的工业标准答案）。

面试收束句：「长上下文与 RAG 不是替代关系，而是同一个稀疏注意力问题在不同层的解；架构进步只是不断把交叉点右移。真正被杀死的不是 RAG，而是『只会切块+向量库』的朴素 RAG。」

附录 A1 · CS224R L1

MDP 与 RL 目标：主线所有目标函数的「量纲」先把「优化的到底是什么」写清楚

\[ \max_\theta\ \mathbb{E}_{\tau\sim p_\theta(\tau)}\Big[\sum_{t=1}^{T} r(s_t,a_t)\Big],\quad p_\theta(\tau)=p(s_1)\prod_t \pi_\theta(a_t|s_t)\,p(s_{t+1}|s_t,a_t) \]
轨迹 \(\tau=(s_1,a_1,\dots,s_T,a_T)\)。三件配套工具：\(V^\pi(s)=\mathbb{E}[\text{未来回报}|s]\)、\(Q^\pi(s,a)\)、优势 \(A^\pi=Q^\pi-V^\pi\)（「这个动作比平均好多少」）。
MDP 要素LLM 后训练里的对应物
状态 \(s_t\)prompt + 已生成前缀 \((x, y_{<t})\)
动作 \(a_t\)下一个 token \(y_t\)（动作空间 = 词表）
转移 \(p(s'|s,a)\)确定性拼接（智能体场景：+工具返回，随机）
奖励 \(r\)通常仅终点：\(r(x,y)\)（验证器/RM），\(\gamma=1\)

⛏️ 挖到主线：主线所有目标函数（GRPO、OPMD、IcePop、PARL）都是这同一个 \(\max\mathbb{E}[\sum r]\) 的不同估计器。「智能体 RL」之所以难，正是因为转移从确定性拼接变回了真随机（环境/工具/子体的返回不可预测）—— MDP 的完整机制苏醒了。

面试追问：为什么 LLM RL 通常 \(\gamma=1\) 且只有终点奖励？→ 序列短（相对机器人的无限时域）、成败只在最终答案可判；也正因此信用分配全靠优势估计——这就是后面所有算法演化的火种。

附录 A2 · CS224R L3

REINFORCE：策略梯度 = 带权重的模仿学习log-derivative trick —— 主线一切 \(\nabla\log\pi\cdot A\) 的出生证明

\[ \nabla_\theta J=\mathbb{E}_{\tau\sim p_\theta}\big[\nabla_\theta \log p_\theta(\tau)\cdot r(\tau)\big] \]
恒等式 \(\nabla p = p\,\nabla\log p\) 把「对期望求导」变成「对 log 概率求导的期望」——转移概率不依赖 \(\theta\)，故 \(\nabla\log p_\theta(\tau)=\sum_t\nabla\log\pi_\theta(a_t|s_t)\)：不需要环境可微！
\[ \nabla_\theta J=\mathbb{E}\Big[\sum_{t}\nabla_\theta\log\pi_\theta(a_t|s_t)\Big(\underbrace{\textstyle\sum_{t'\ge t} r_{t'}}_{\text{reward-to-go}}-b\Big)\Big] \]
因果性：\(t\) 时刻的行为影响不了 \(t\) 之前的奖励 —— 只对未来奖励加权，白降方差不引偏。

课程里最美的直觉：把它与模仿学习的梯度 \(\mathbb{E}[\nabla\log\pi(a|s)]\) 并排看 —— 策略梯度就是「按奖励加权的模仿学习」：多模仿高分轨迹里的自己，少模仿低分轨迹里的自己。「试错」第一次有了微分形式。

⛏️ 挖到主线：实现上用「替代目标」\(\tilde J=\frac1N\sum\log\pi_\theta\cdot(\hat R-b)\)（加权极大似然）——你现在能看懂为什么 GRPO/K2.5/MAI 的目标函数都长成「\(\log\pi\) 或比率 × 优势」：它们全是这个替代目标加上不同的方差与漂移控制。K1.5 的平方损失求导后也回到这里。

课程警句：原始策略梯度噪声极大、对奖励尺度敏感、最好配大 batch 与稠密奖励 —— 这三句话预言了主线的全部改进方向。

附录 A3 · CS224R L3/L4

基线与优势估计：方差工程的完整谱系从「减去均值」到 GAE，再到「干脆用组内统计」

\[ \mathbb{E}_{a\sim\pi_\theta}\big[\nabla_\theta\log\pi_\theta(a|s)\cdot b\big]=b\,\nabla_\theta\!\!\sum_a\pi_\theta(a|s)=b\,\nabla_\theta 1=0 \]
一行证明：减去任意常数基线不改变梯度期望（无偏），却能大幅削方差 —— 平均奖励就是很好的基线。
\[ \hat A^{\mathrm{GAE}}_t=\sum_{l\ge0}(\gamma\lambda)^l\,\delta_{t+l},\qquad \delta_t=r_t+\gamma V(s_{t+1})-V(s_t) \]
价值函数的两种拟合：MC 回归（无偏高方差）vs TD 自举（低方差有偏）；GAE 用 \(\lambda\) 在两端连续调和。这是 PPO（含 Tülu 3 的 RLVR）的标准配置。

⛏️ 挖到主线：2025 年的关键工程判断 —— 在「终点 0/1 奖励 + 超长序列」下，critic 既学不准也太贵，于是 GRPO 用「同题多解的组内均值/方差」顶替 critic：基线从「学出来的函数」退回「算出来的统计量」，无偏性由上面那行证明背书。MAI/K2.5 的 token 级归一化则是把「哪些 token 分到多少信用」这个 GAE 未竟的问题，用等权重粗暴而稳健地回答。

面试追问：既然 GRPO 赢了，价值函数死了吗？→ 没有。过程奖励/长程信用分配（主线综合篇的开放问题）大概率要它回归 —— 只是形态可能是「隐式价值」（如 IQL 式，见附录 B2）而非 PPO 的显式 critic。

附录 A4 · CS224R L3/L5/L6

重要性采样 → KL 约束：off-policy 的原罪与救赎主线所有「裁剪/门控」的数学出生地

\[ \nabla J(\theta')=\mathbb{E}_{\tau\sim p_\theta}\Big[\prod_{t=1}^{T}\frac{\pi_{\theta'}(a_t|s_t)}{\pi_\theta(a_t|s_t)}\cdot(\cdots)\Big] \]
想复用旧策略的数据 → 重要性采样。但轨迹级比率是 \(T\) 个比率的连乘：长序列下爆炸或消失。
\[ \nabla J(\theta')\approx\frac1N\sum_{i,t}\frac{\pi_{\theta'}(a_{i,t}|s_{i,t})}{\pi_\theta(a_{i,t}|s_{i,t})}\,\nabla\log\pi_{\theta'}\cdot\hat A_{i,t},\quad \text{s.t. } \mathbb{D}_{\mathrm{KL}}(\pi_{\theta'}\|\pi_\theta)\le\varepsilon \]
救赎两步：①退到 token 级比率（状态分布比率近似为 1）；②加 KL 信任域限制单步偏移。PPO 的 clip 就是这个约束的一阶便宜实现。

⛏️ 挖到主线：把这页当「解码器」用 —— GRPO 的 clip、GLM 的 IcePop 门控、K2.5 的 log-ratio 双侧裁剪、MAI 的自适应熵上界，全是「token 级比率 + 某种信任域」这一个模板的四种填空。区别只在比率对谁算（旧策略？推理引擎？）和越界后干什么（截断值？置零梯度？）。

面试追问：为什么 LLM 后训练不用 Q-learning/replay buffer 这套「真 off-policy」？→ ①动作空间 = 词表、序列组合爆炸，\(\max_a Q\) 不可算；②终点稀疏奖励下 TD 自举要跨数千 token 传播，误差累积；③预训练已给出极强先验策略，policy-based 微调远比 value-based 从头估值高效。课程里 SAC/DQN 一脉在 LLM 世界只剩「轻度 off-policy + 快拉回」的折中形态。

附录 B1 · CS224R L2

模仿学习：SFT 的真名与复合误差Behavior Cloning、covariate shift、DAgger —— OPD 的祖父

\[ \min_\theta\ -\,\mathbb{E}_{(s,a)\sim\mathcal D}\big[\log\pi_\theta(a|s)\big] \]
BC = 在专家数据上做生成式建模。SFT 就是 BC，「自回归分布 = 最大表达力的动作分布」——LLM 天生持有课程里最强的策略类。
  • 复合误差：监督学习假设输入独立于预测；但策略的动作决定下一个状态 → 小错把自己带出数据分布 \(p_{\text{expert}}(s)\neq p_\pi(s)\) → 错上加错；
  • DAgger：跑自己的策略、在自己访问的状态上要专家标签、聚合再训 —— 把误差从 \(O(\varepsilon T^2)\) 压回 \(O(\varepsilon T)\)；
  • HG-DAgger：专家只在出错时接管（干预式数据）。

⛏️ 挖到主线（一箭三雕）：①LLM 的暴露偏差就是复合误差的换名；②OPD 是 DAgger 的工业化身——「在学生自己的轨迹上向教师要逐 token 标签」与 DAgger 的伪代码逐字对应，只是专家从人变成了教师模型（查询成本→0，DAgger 的最大痛点消失了）；③GLM-5「错误片段保留但屏蔽 loss」= 干预式纠错数据的 LLM 形态。

面试追问：SFT 数据越多越好吗？→ 课程答案：offline BC 无论多少数据都受分布外漂移威胁，「可靠性来自在自己的分布上获得纠正」—— 这正是为什么 2026 所有旗舰把资源从 SFT 挪向 on-policy 的 RL 与蒸馏。

附录 B2 · CS224R L7

离线 RL：从固定数据集里挤出超越数据的策略Filtered BC → AWR → IQL —— 以及它与 DPO / 拒绝采样 / 搜索日志的血缘

核心病灶：静态数据集上做 Q 学习，策略会追逐 OOD 动作上被高估的 Q 值（网络在数据支撑集外自由幻想）。三级解药：

\[ \text{①过滤式 BC：只模仿 } r(\tau)>\eta \text{ 的轨迹} \] \[ \text{②AWR：}\ \theta\leftarrow\arg\max_\theta\,\mathbb{E}_{\mathcal D}\big[\log\pi_\theta(a|s)\,e^{A(s,a)/\alpha}\big] \] \[ \text{③IQL：}\ V\leftarrow\arg\min\ \mathbb{E}\big[\ell_2^\lambda\big(V(s)-Q(s,a)\big)\big] \]
AWR 近似「KL 约束下最大化 Q」的闭式解 \(\pi^*\!\propto\!\pi_\beta e^{A/\alpha}\)；IQL 用期望位数回归（不对称平方损失）让 \(V\) 逼近数据支撑内动作的高分位值——全程不查询任何 OOD 动作。离线 RL 能做「轨迹拼接」：把两段平庸轨迹的好段落缝成超越任何示范者的路径。

⛏️ 挖到主线：①拒绝采样微调 / best-of-n 蒸馏（R1 第三阶段、SWE-smith 的 RFT）就是过滤式 BC；②AWR 的 \(\pi^*\!\propto\!\pi_\beta e^{A/\alpha}\) 与 DPO/K1.5 的 \(\pi^*\!\propto\!\pi_{\mathrm{ref}}e^{r/\beta}\) 是同一个闭式解——DPO 本质是离线偏好 RL；③「别信数据支撑集外的值」正是 KL 正则/裁剪的离线版表述。

⛏️ 挖到搜索系统：企业里最大的数据资产是用户交互日志——天然的离线 RL 数据集（行为策略=旧版模型+用户混合）。课程的警告直接适用：拿日志做 naive RL 会追逐日志外动作的幻想收益；正解是 AWR/IQL 式保守学习，或把日志用于训 GRM/校准（落地篇①的立场，在此获得理论根据）。

词向量 → 稠密检索：召回层的数学word2vec 的对比学习基因，一路遗传到 DPR 双塔

面试追问：为什么不用生成器的嵌入直接做检索？→ 检索需要「对称、可预计算、内积可索引（ANN）」的表征；生成模型的表征为下一 token 优化，二者目标不同 —— 这就是嵌入模型独立存在（并被教师蒸馏，见落地③）的原因。

附录 C2 · CS224N L10

RAG 的证据链：FiD、迷失在中间、幻觉引用课程实验数据，恰好是落地篇两大设计决策的证词

  • Retriever–Reader → FiD：从「抽取答案」到「融合多段生成答案」（DPR+T5，各段独立编码、解码器统一融合）—— 现代 RAG 生成器的原型；
  • Lost in the middle（Liu+ 2023）：相关文档放在上下文中部时性能骤降 —— LM 的注意力并不均匀；
  • RAG 饱和：检索器召回率随文档数缓慢升到 90%，但 RAG 端到端性能在 10–20 篇后即饱和 —— 塞更多文档≠更好答案；
  • 幻觉引用：引用本身也是生成的——课程实验：输出「看起来有用」（人评高分）但引用的精确率与召回率都很低。

⛏️ 挖到主线与落地篇：①「10–20 篇饱和 + 迷失在中间」正是长上下文 vs RAG 交叉点存在的微观机理——V4/GLM-5.2 的压缩注意力（可学习的「注意什么」）是对它的架构级回答，检索粗筛+精读是系统级回答；②「幻觉引用」实验为落地①的引用可验证奖励提供了必要性证明：引用格式正确≠引用真实支持，所以需要 NLI 蕴含判定 + 两级免疫。课程指出病，主线论文开的方。

面试追问：top-k 取多大？→ 不是越大越好：先给召回率-饱和曲线的定性答案（20 内），再谈重排后精选 3–5 篇 + 位置工程（关键证据放两端），最后提 agentic 逐步检索绕开单轮上下文限制。

附录 C3 · CS224N L10

语言智能体的学术前身：ReAct、Reflexion、记忆、Toolformer2022–2024 的这些「小」论文，是主线智能体 RL 的概念货架

  • ReAct：交替「思考↔行动」——推理是内部动作，为观察→动作的艰难映射搭桥；显著减少 CoT 幻觉；
  • Reflexion：失败轨迹→语言化自我反思→存为提示 —— 用自然语言实现的「无梯度 RL」；
  • 记忆三分类：情景（事件流）/语义（提炼知识）/程序（代码技能, Voyager）——读=检索，写=追加或反思；
  • Toolformer：自监督插入 API 调用——只保留能降低后续 token 损失的调用；
  • 多智能体辩论 / Orchestrator（Magentic-One）：提示工程时代的组织结构。

⛏️ 挖到主线（每条都有直系后代）：ReAct → GLM「交错思考」与 K2 的思考-调用交替（从提示技巧变成 SFT 模板 + RL 目标）；Reflexion 的语言反馈 → K2 自评 rubric / GRM（从提示存档变成训练信号）；记忆分类 → K2.5 Swarm 的「子体只回传引用+摘要」与 DeepResearch 上下文管理；Toolformer 的「按对 loss 的贡献筛选调用」→ K2 工具合成流水线的 judge 过滤；辩论/Orchestrator → PARL 把「组织结构」本身变成可训练策略。一句话：2023 年靠提示实现的一切，2026 年都被后训练内化成了参数。

课程同页彩蛋：SWE-smith 用「拒绝采样微调（RFT）」训编码智能体 —— 又一次遇见过滤式 BC（附录 B2）。

附录 D1 · CS224R L8

偏好奖励学习：Bradley–Terry 与 2017 年的先知Christiano et al. —— RLHF 在成为 LLM 技术之前，是机器人技术

\[ P(\tau^1\succ\tau^2)=\sigma\Big(\sum_t r_\phi(s_t^1,a_t^1)-\sum_t r_\phi(s_t^2,a_t^2)\Big) \] \[ \mathcal L(\phi)=-\,\mathbb{E}_{(\tau^1\succ\tau^2)}\big[\log P(\tau^1\succ\tau^2)\big] \]
人类只比较轨迹片段对（比打绝对分容易且一致），BT 模型把比较转成奖励差的极大似然。2017 年用约 900 次人类比较教会模拟 Hopper 后空翻 —— 五年后同一配方 + LLM = InstructGPT。

逆强化学习（IRL）一句话：奖励是对行为的解释而非给定——从示范反推奖励。它提醒我们：主线里「验证器/rubric/GRM」都是在人为指定解释，指定就会有漏洞。

⛏️ 挖到主线：①DPO 的推导正是把这页的 BT 塞进 KL 约束闭式解（附录 B2 的同一个 \(\pi^*\)）—— 三条线索在此汇合；②奖励作弊的理论根源在这页就种下：\(r_\phi\) 是代理，Goodhart 定律说「指标被优化就不再是好指标」——主线的 rubric 轮换、两级免疫、行为信号低权重化，全是对同一条定律的工程回应。

面试追问：为什么偏好对比而不是绝对打分？→ 人类打绝对分噪声大、标尺漂移；比较是序数判断，跨标注员一致性高——同理，K2 自评 rubric 用两两排序而非打分，GRM 输出多维评价而非单一分数。

附录 D2 · CS224N L4/L14

解码与分词：rollout 工程的两块底砖采样策略决定探索；分词决定「动作空间」本身

\[ p_i\propto\exp(z_i/T);\qquad \text{top-}p:\ \text{最小集合 } S \text{ 使 } \textstyle\sum_{i\in S}p_i\ge p \]
解码谱系：greedy/beam（确定性，易重复）→ 温度 \(T\)（整体锐化/平滑）→ top-k / top-p（截断尾部幻觉区）。RL 视角：解码策略 = 探索策略。
  • BPE 分词：从字符对频率自底向上合并——词表是学出来的压缩码本；
  • 同一段文本可有多种 token 化 ⇒ 文本往返（text-in-text-out）会造成重分词漂移。

⛏️ 挖到主线：①K2 的温度衰减 = 「先探索后利用」在解码参数上的实现——课程的采样谱系给了它坐标；②GLM-5 异步 RL 坚持 token-in-token-out，正是因为重分词漂移会污染重要性比率（附录 A4 的失配又多了一个来源——这次来自分词器）；③投机解码（GLM-5.2 +20% 接受率）本质是「便宜策略提议、昂贵策略验证」——与 PARL 的「小子体先训、大子体后换」共享同一种成本结构直觉。

面试追问：RL 训练时 rollout 温度怎么定？→ 答题框架：太低→组内全同、GRPO 优势为零、无梯度；太高→劣质样本淹没信号。所以「初期高温+衰减」（K2）或熵目标闭环（MAI）——把温度从超参变成被控变量。

附录 D3 · CS224N L11

评测方法论：整条飞轮的「度量衡署」测不准的能力必然被作弊 —— 评测是奖励设计的镜像问题

  • 污染：评测集混入训练数据 → 榜单虚高。对策：n-gram/嵌入去污染（Tülu 3 的纪律）、私有 unseen 套件（MAI 的内部 NLL 集）、动态换题（AIME 逐年、LiveCodeBench 滚动）；
  • LLM-as-judge 的系统性偏差：位置偏好（先出现的赢）、长度偏好（长=好）、自恋偏好（偏爱同家族文风）——对策：换序重评、长度控制、多裁判；
  • 智能体评测的三难：环境搭建复杂、成功标准开放、多条有效路径 —— 不能只看 pass rate，还要审计过程(步数/成本/违规)。

⛏️ 挖到主线：评测和奖励是同一个函数的两次使用（离线叫评测，在线叫奖励），所以它们生同样的病：judge 偏差 ↔ GRM 被讨好；污染 ↔ 训练泄漏测试用例（GLM-5.2 的 curl 作弊正是「主动自我污染」）。主线的答案一体两面：评测侧去污染+多裁判，奖励侧 rubric 轮换+两级免疫。

⛏️ 挖到搜索系统：落地篇全景图里的「评测闸门」由此页展开——企业版三件套：去污染的内部问答金标集（防吃进训练）、带位置/长度校正的 GRM 抽检、DeepResearch 的过程审计（引用真实率/步数预算）。没有可信度量衡，飞轮转得越快错得越远。
