为什么是漏斗

算力守恒：对十亿候选跑重模型不可行。于是召回用最便宜的方法处理最多候选，精排用最贵的模型处理最少候选——这个"级联漏斗"约束了后面所有技术的形态：任何新模型想上线，都必须回答"你打算住进哪一层、付得起那层的延迟房租吗"。

第一幕 · Unicorn（Facebook, VLDB 2013）

搜索 = 集合运算 Search as Set Operations

直觉

把 Facebook 社交图谱想象成一座图书馆的索引卡片柜：每张卡片是一个属性（"是 David 的朋友""住在纽约"），卡片后挂着满足该属性的实体列表（posting list）。搜索一句话 = 把几张卡片叠起来做交并差。图遍历也能用倒排索引的语言表达——这是 Unicorn 的核心洞察。

数学：集合代数
$$\text{result} = \mathrm{PL}(t_1)\;\square\;\mathrm{PL}(t_2)\;\square\;\cdots,\qquad \square\in\{\cap,\,\cup,\,\setminus\}$$

\(t_i\) 是前缀化词条，如 friend:10003（10003 号用户的好友）。\(\cap/\cup/\setminus\) 对应 and / or / difference 算子。

$$\mathrm{apply}(E,\;\text{edge\_type}) = \bigcup_{v\in E}\mathrm{PL}(\text{edge\_type}:v)$$

apply = 多跳魔法：先算出集合 \(E\)（"我在纽约的朋友"），再把每个结果当新种子查下一类边——"我在纽约的朋友的雇主"这种二跳查询就此成立。

排序靠 static rank（与查询无关的重要性分），让每条 posting list 支持"取 top-K 而非全量遍历"。

在骨架中的位置

占据② 召回的地基：把"图遍历"改造成"倒排索引上的集合运算"，统一取代了 Facebook 内部多套关键词搜索后端。static rank 是③排序的雏形，查询树解析是①查询理解的早期形态。

符号时代的世界观

相关性是离散、精确、可解释的：一个文档要么包含词条、要么不包含；一个人要么是你朋友、要么不是。这套世界观让系统极快、极可控——但它有一个致命盲区，下一页揭晓。

第一幕 · 幕间 · 符号时代的天花板

语义鸿沟 The Semantic Gap

你搜"讲鬼故事的地方"，而完美答案叫"万圣节恐怖夜活动"——一个字都不重合，倒排索引永远召回不了它。

问题的本质

倒排索引把"相关"定义为字面命中（token 精确匹配）。但人类语言里，同一个意图有无数种表达（同义、缩写、多语言、错拼、隐喻）。词汇不匹配（lexical mismatch）不是长尾问题——它是召回环节的系统性盲区。

补丁方案（同义词表、查询扩展规则）治标不治本：规则永远追不上语言的组合爆炸。

需要一次世界观革命

如果"相关性"不再定义为字面命中，而是定义为某个连续空间中的距离呢？

第二幕 · Facebook EBR（KDD 2020, arXiv:2006.11632）

相关性 = 空间距离 Embedding-Based Retrieval

直觉

把 query 和 document 都映射进同一个向量空间，让语义相近的东西靠得近。检索从"字符串匹配"变成"找最近的邻居"——即使一个字都不重合，只要语义近，也能被召回。语义鸿沟就此跨越。

为什么是"双塔"
$$D(q,d) = 1-\cos\big(u(q),\,v(d)\big)$$

query 塔 \(u(\cdot)\) 与 document 塔 \(v(\cdot)\) 各自独立编码、互不相见——这不是精度上的最优，而是工程上的必然：只有独立编码，document 向量才能全部离线预计算、建成索引；在线只需算一次 query 向量 + 一次近邻查找。

统一嵌入 Unified Embedding

EBR 的"统一"指输入不只是文本：query 塔吃进搜索者位置、社交图谱嵌入等上下文；document 塔同理。社交搜索的相关性天然是"文本 + 人 + 场景"的函数。

训练目标：Triplet Loss
$$L=\sum_{i=1}^{N}\max\Big(0,\;D(q^{(i)},d^{(i)}_+)-D(q^{(i)},d^{(i)}_-)+m\Big)$$

\(d_+\)：点击过的正例；\(d_-\)：负例；\(m\)：margin 间隔。
它在优化什么：让正例距离比负例至少近 \(m\)。一旦差距超过 \(m\)，该样本梯度归零、不再学习——模型注意力被自动集中到难分的样本上。

margin 是极敏感的超参数：取值不同可造成 5%–10% 的 KNN recall 波动。

第二幕 · EBR 最重要的工程洞察

难负例为何关键 Hard Negative Mining

全文最值得背下来的一页：负例的分布，必须匹配线上检索时真实面对的候选分布。

两个失败的极端

只用"曝光未点击"当负例 → 训练集里全是"字面沾边但没被点"的难例，而线上候选池 99% 是毫不相关的易例——训练/服务分布错配，recall 显著变差。

只用随机负例 → 模型只需学会"字面搭不搭边"就能把损失降到很低，永远学不会区分"看起来沾边但不该召回"的高相似候选。

EBR 的配方
  • 在线挖掘：batch 内借用别的 query 的正例当难负例（每条取 k=2 最优）。
  • 离线挖掘：用上一版模型对全库做 ANN，取排名 101–500 名当难负例——不取最难的前几名，因为那里常是漏标的假负例。
  • 混合比例：易 : 难 ≈ 100 : 1 效果最好。

第二幕 · 让向量召回跑进毫秒级

ANN 为何可行 + 全栈混合检索

HNSW：多层小世界图

把向量组织成多层图——上层稀疏、边长（像坐飞机跨城），下层稠密、边短（像市内公交）。查询从顶层贪心走向更近邻居，逐层下沉细化。小世界性质保证 \(O(\log N)\) 步就能逼近目标，避免线性扫描。

IVF-PQ：先分街区，再查表打分

IVF 粗量化：k-means 把全库聚成数千簇（分"街区"），查询只进最近的几个簇。
PQ 乘积量化：把高维向量切成子段、各自用小码本压缩成几个字节，距离计算变成查表——内存与算力同时骤降。EBR 使用 Faiss 的粗量化 + OPQ 组合。

共同哲学：用可控的精度损失，换指数级的速度提升。

第二幕 · DPR（Karpukhin et al., EMNLP 2020, arXiv:2004.04906）

In-batch Negatives：免费的午餐 Dense Passage Retrieval

直觉

与 EBR 同年、殊途同归的学术里程碑：只用几万条问答对训练双塔，就在开放域问答上击败打磨了几十年的 BM25。它的训练技巧朴素到惊人——同一个 batch 里，别人的正确答案就是我的负例，无需任何负例挖掘流水线。

数学
$$\mathrm{sim}(q,p)=E_Q(q)^{\top}E_P(p)$$
$$L=-\log\frac{\exp\big(\mathrm{sim}(q_i,p_i)\big)}{\sum_{j=1}^{B}\exp\big(\mathrm{sim}(q_i,p_j)\big)}$$

它在优化什么：把检索变成 batch 内的多分类——"这 \(B\) 个段落里谁是我的"。一次前向传播免费获得 \(B-1\) 个负例，batch 越大负例越丰富。实际训练 B=128，再混入 1 个 BM25 难负例提升区分度。

在骨架中的位置

改造开放域 QA 的②召回：稠密向量取代 BM25，成为下游 Reader 的供给。这条 Retriever → Reader 链路正是第三幕 RAG 范式的直接前身——记住它。

第二幕 · Que2Search（Facebook, KDD 2021）

给双塔换脑子 XLM-R · 多模态 · 课程学习

直觉

EBR 用"字符 n-gram + 浅层网络"近似语义，对短查询、多语言、带图的电商场景（Marketplace）力不从心。Que2Search 把词袋换成跨语言预训练模型 XLM/XLM-R，把商品图片（GrokNet）也编进同一空间，并用注意力对多路特征做可学习的融合：

$$w=\mathrm{softmax}\big(\mathrm{proj}(\psi_1\Vert\cdots\Vert\psi_N)\big),\qquad f=\sum_i w_i\,\psi_i$$

短 query 更依赖 3-gram、长 query 更依赖 XLM——通道权重 \(w\) 让模型自己决定信谁。

两阶段课程学习
$$\text{阶段一：}\; L_i=-\log\frac{e^{s\cos(q_i,d_i)}}{\sum_{j=1}^{B}e^{s\cos(q_i,d_j)}}$$
$$\text{阶段二：}\; n^{q_i}=\arg\max_{j\neq i}\cos(q_i,d_j),\;\; L_i=\max\big(0,\,\cos(q_i,d_{n^{q_i}})-\cos(q_i,d_i)+m\big)$$

先用 batch 内负例的 softmax（缩放系数 s≈15–20）训到收敛，再挖 batch 内最难负例用 margin loss 精调（m≈0.1–0.2）。顺序不可颠倒——先易后难，正是"课程"二字的含义，也治好了 EBR 难负例调参易翻车的痛点。

架构与多任务

Query 塔：3-gram 哈希 + 国家 embedding + 2 层 XLM，CPU 上 P99 延迟 1.5ms 以内（在线实时）；Doc 塔：6 层 XLM-R + 图片向量（多图用 Deep Sets 聚合，离线预计算）。额外的多任务头：用 doc 向量预测"它历史上被哪些 query 搜到过"（4.5 万类多标签）——逼模型学到商品与搜索意图的本质关联。

在骨架中的位置

同时触及三个环节：①查询理解（XLM 语义表示）、②召回（接入 Unicorn ANN）、③排序（cosine 作为 GBDT 粗排与深度精排的强特征）。预训练语言模型自此正式入栈——第三幕 LLM 全面接管的前奏。

第二幕 · 三种范式的光谱

双塔 · 交叉编码器 · ColBERT 速度与精度的光谱

ColBERT 的 MaxSim
$$S_{q,d}=\sum_{i=1}^{N}\max_{j=1..M}\;E_{q_i}\cdot E_{d_j}^{\top}$$

每个 query token 在文档所有 token 里找最佳落脚点（max），再求和。文档端 token 向量离线预计算——保住词级交互的精度，摊掉重复前向的开销。比 BERT 交叉编码器快两个数量级、省四个数量级 FLOPs，MS MARCO MRR@10 ≈ 0.36（与 CE 相当）。

这组权衡决定了骨架的形状

正因为"能预计算的不够准、够准的不能预计算"，骨架必须是级联漏斗：双塔/倒排管召回（快而粗），交叉编码器管精排（慢而准），ColBERT 插在中间两头通吃——它既能当高效精排器，也因剪枝友好可下沉做端到端召回。

级联漏斗成型：骨架的 2021 版

第三幕 · 主题一 内容理解 3/3 · Semantic ID / TIGER

把内容压成离散基因编码 RQ-VAE 语义 ID

直觉

传统 item ID 是不透明数字，模型对它零先验，新内容冷启动学不到东西。Semantic ID 给每个内容一串"基因编码"——把 embedding 层层压缩成几个离散数字（如 12-87-45），相似内容共享前缀、后缀微调。生成式模型从此可以"逐位写出"要检索的内容，而不是在亿级 ID 上做多分类。

RQ-VAE：残差量化
$$c_i=\arg\min_k\lVert r_{i-1}-e_k^i\rVert^2,\qquad r_i=r_{i-1}-e_{c_i}^i$$
$$\mathcal{L}=\lVert z-\hat z\rVert^2+\sum_{i=1}^{m}\Big(\lVert \mathrm{sg}[r_{i-1}]-e_{c_i}^i\rVert^2+\beta\lVert r_{i-1}-\mathrm{sg}[e_{c_i}^i]\rVert^2\Big)$$

像十进制逐位逼近一个连续数：第一位粗定位，每一位在上一位的残差里继续细分——层级天然由粗到细，语义层次免费获得。\(\mathrm{sg}\)=stop-gradient，\(\beta\)=承诺损失权重。

一查多发 从"改写一个查询"到"生成一组查询"

直觉

接到模糊的大任务，先在心里拆成子任务清单、多路同时调查、最后汇总成报告。Query fan-out 就是把这个过程机械化：一个复杂 query 被 LLM 分解为一组覆盖不同侧面的子查询，并发检索后由生成模型归并。这不再是"单点优化"，而是查询理解的"多点并发"。

indexing 用"文档全文"、retrieval 用"短查询"，两种输入分布差异巨大。DSI-QG 用查询生成模型给每篇文档造伪查询，用"伪查询→docid"代替"文档→docid"做索引样本——后来被规模化实验证实是生成式检索最关键的组件。

NCI 与 SEAL 的两条岔路

NCI：给解码器装"前缀感知"能力（PAWA），按已生成的 docid 前缀动态调整输出分布 + 查询生成增强（NQ320k Recall@1 相对 +21.4%）。
SEAL：干脆不设计 docid——用文档里真实出现的 n-gram 当标识符，FM-index 约束解码保证只生成语料中存在的片段，再映射回原文档（KILT 上 R-precision 平均高 10 点以上）。

第三幕 · 主题四 生成式检索 3/3 · 现实检验

百万文档的冷水，与推荐系统的暖流

冷水：规模化实验（Pradeep et al., EMNLP 2023）

首次在 MS MARCO 全量 880 万段落上系统评测生成式检索（模型测到 11B），三条结论：

  • 合成查询是核心组件——DSI-QG 式伪查询被反复验证为唯一稳定有效的文档表示。
  • 此前各种架构改动，扣除算力成本后基本无效。
  • 单纯堆参数收益有上限，无法像 LLM 那样持续 scaling。

结论：小语料上可与 SOTA 双塔竞争，百万级以上仍是未解难题。另有 DSI++ 指出：持续加新文档会引发灾难性遗忘——"背书的人换了新书得重背"，需用伪查询回放缓解。

与双塔的权衡

双塔：语料更新只需增量插向量，十亿级稳健；DSI：更新 = 重训模型。网页搜索（万亿级、分钟级更新）暂时无解，但候选集有限、语义结构强的场景正好可行——于是它在推荐系统开花。

第三幕 · 主题五 RAG 1/3 · REALM 与 RAG（2020）

开卷考试 检索作为隐变量

直觉

参数化模型把知识"死记硬背"进权重：陈旧、不可溯源、无法局部更新。RAG 给模型配一个随时可翻的资料库——先检索、再"看着资料作答"。开卷考试：书可以换新版，答题的手还是自己的手。

REALM：检索即隐变量
$$p(y\mid x)=\sum_{z\in\mathcal Z}p(y\mid z,x)\;p(z\mid x),\qquad p(z\mid x)=\frac{e^{f(x,z)}}{\sum_{z'}e^{f(x,z')}}$$

\(z\)=从语料库挑出的文档（隐变量），\(f(x,z)\)=双塔内积。关键：梯度穿过检索概率 \(p(z|x)\) 回传——检索器不是外挂模块，而是随语言建模目标一起被训练。工程难点：文档向量随参数更新而过时 → 异步重建 MIPS 索引（每几百步刷新一次）。

RAG：两种边缘化粒度
$$p_{\text{RAG-Seq}}(y\mid x)\approx\sum_{z\in\text{top-}k}p_\eta(z\mid x)\prod_{i=1}^{N}p_\theta(y_i\mid x,z,y_{1:i-1})$$
$$p_{\text{RAG-Tok}}(y\mid x)\approx\prod_{i=1}^{N}\sum_{z\in\text{top-}k}p_\eta(z\mid x)\,p_\theta(y_i\mid x,z,y_{1:i-1})$$

Sequence：选定几篇资料通篇写完（求和在外）；Token：每写一个词都可换资料看（求和在内）——粒度更细、计算更贵。检索器 \(p_\eta\) 就是 DPR 双塔：第二幕的召回技术原封不动成为 RAG 的地基。

+4~16pp
REALM 在三个 Open-QA 基准上的绝对提升
SOTA
RAG 三个 ODQA 任务当时最优，生成更具体、更符合事实

第三幕 · 主题五 RAG 2/3 · FiD 与 Self-RAG

翻书的工程学与自觉性

FiD：解码器侧融合（翻书效率工程）
$$\mathbf H_i=\mathrm{Enc}\big([q;p_i]\big)\;\;(i=1..k),\qquad y=\mathrm{Dec}\big(\mathrm{Concat}(\mathbf H_1..\mathbf H_k)\big)$$

编码器独立并行编码每篇"问题+段落"（线性开销），融合动作推迟到解码器的交叉注意力一次完成。若把 k 篇拼成长序列整体编码，自注意力是平方级开销——FiD 因此能吃下几十篇文档而显存不爆，NQ/TriviaQA 当时 SOTA，且效果随段落数持续上涨。

位置速记

REALM/RAG/FiD 共同确立：④结果呈现从"十条蓝链"变成"生成答案"，而②召回③排序整条链路被 RAG 原样复用——检索不到，就编不出真话；检索质量仍是答案质量的上限。

Self-RAG：把"要不要翻书"也交给模型
$$p(y_t,\,r_t\mid x,\,y_{\lt t})\qquad r_t\in\{\texttt{Retrieve},\texttt{ISREL},\texttt{ISSUP},\texttt{ISUSE}\}$$

词表里加入四类反思 token，与正文一起被自回归生成：
Retrieve＝这步要不要查资料？ISREL＝查到的段落相关吗？ISSUP＝我写的话有依据吗？ISUSE＝整段回答有用吗？
检索从"流程写死的每次 top-k"变成模型自主决策的动作；反思分数还用于 segment 级 beam search 挑选最优延续。

7B/13B
Self-RAG 在多任务上超过 ChatGPT 与带检索的 Llama2-chat，长文本事实性与引用准确率显著提升

留意 ISREL / ISSUP 的本质：模型在给检索质量和事实一致性"打分"——这已经是内嵌在生成过程里的 AI Judge。下一个主题的伏笔再次出现。

第三幕 · 主题五 RAG 3/3 · Agentic RAG / Deep Research（2025–26）

从翻一次书，到带着问题跑图书馆 Agentic RAG

直觉

RAG 1.0 是"查一次、答一次"；Deep Research 是研究员工作流：先写研究计划 → 多轮"搜索-阅读-推理"循环 → 判断信息够不够 → 不够再查 → 最后写出带引用的报告。检索从一次函数调用，变成智能体循环里反复出现的动作。

何谓"真正 Agentic"（Survey, arXiv:2501.09136）
  • 自主策略：LLM 自己选择并组织检索/推理策略。
  • 迭代执行：根据中间结果动态调整的多轮循环。
  • 交织工具调用：类 ReAct 的"行动-观察"交替。
对骨架的意义

Agentic RAG 把①查询理解（分解子问题）、②召回（多轮检索）、④呈现（综合报告）重新揉进一个循环——骨架清晰的阶段边界第一次被模糊。它不替换骨架，而是站在骨架之上反复调用骨架。

第三幕 · 主题六 AI as a Judge 1/3 · Thomas et al.（Bing, SIGIR 2024）

AI 标注员上岗 LLMs Can Accurately Predict Searcher Preferences

直觉

传统相关性标注靠雇佣第三方标注员——但标注员不是真实用户，常常标不到点子上。Bing 反其道行之：先收集小规模、高质量的真实用户反馈当"北极星"，再反复调 LLM 的 prompt，直到它的判断与真实用户对齐——训练的不是模型，是提示词；校准好后即可无限复制这位"最懂用户的标注员"。

数学：prompt 即参数
$$\text{Prompt}^*=\arg\max_{\text{Prompt}}\;\mathrm{Agreement}\big(\mathrm{LLM}_{\text{Prompt}}(q,d),\;y^*\big)$$
$$\kappa=\frac{p_o-p_e}{1-p_e}$$

\(y^*\)=真实用户黄金反馈；一致性用 Cohen's \(\kappa\)（\(p_o\)=观测一致率，\(p_e\)=随机期望一致率）。LLM 零训练（zero-shot）——搜索质量评估的"参数"变成了一段自然语言。

→

→

→
大规模离线标注成本仅人工的一小部分
最关键的一句话

论文明确报告：LLM 标注质量与人类标注员相当，且用 LLM 标签训出的排序器"明显更好"（notably better rankers）。也就是说——Judge 的产出不止用于"打分"，而是直接进入训练数据池，成为主题三蒸馏链路最上游的教师标注来源。评估与训练在此合流。

第三幕 · 主题六 AI as a Judge 2/3 · UMBRELA 与 RAGAS

开源复现与生成时代的新考卷

UMBRELA：把秘方变菜谱

Bing 方案未开源。UMBRELA 用 GPT-4o + DNA prompt（理解意图 → 匹配度 → 可信度 → 0-3 分）复现并开源，被 TREC 2024 RAG Track 正式采用。核心发现（DL19-23）：

指标数值解读
Cohen's κ（单条标签）0.31–0.37逐条一致性只算中等
Kendall τ（系统排名）0.87–0.94用 LLM 标注判断"哪个系统更好"，结论与人类高度一致
Spearman ρ（系统排名）0.97+

洞察：Judge 不必逐条复刻人类判断，只要在"系统级排名"这个更粗、更实用的粒度上可靠即可——这正是工业迭代真正需要的粒度。

RAGAS：评估对象从排序变成生成
$$\text{Faithfulness}=\frac{\#\{\text{答案中被 context 支持的 claim}\}}{\#\{\text{答案中全部 claim}\}}$$
$$\text{AnswerRel}=\frac{1}{n}\sum_{i=1}^{n}\cos\big(E(q),\,E(q_i)\big)$$

Faithfulness：把回答拆成可验证的陈述，逐条对照检索内容查"有没有编"。
Answer Relevance 的反推巧思：让 LLM 从答案反推"它最适合回答什么问题"\(q_i\)，与原问题算相似度——切题的回答应能反推回原问题。
外加 Context Relevance（检索内容里有用句子占比）。三项全由 LLM 现场判定，无需人工 ground truth，可当 CI 一样嵌进 RAG 迭代流程。

三大偏差（MT-Bench, NeurIPS 2023）
  • 位置偏差：交换两个答案的呈现顺序，判决可能翻转——检测法：正序逆序各问一次，统计翻转率。
  • 冗长偏差：话越长越像认真作答，得分越高。
  • 自我偏好：GPT-4 裁判偏爱"读起来像 GPT-4"的答案（与文本困惑度更低有关，2025-26 持续有量化与校准工作）。

参照系：GPT-4 judge 与人类一致率 约 85%，而人类-人类一致率约 81%——足够可靠，但绝非无偏。校准（换序取平均、分布重标定）是闭环前的必修课；2024 年亦有工作明确论证"LLM 标注仍不能完全替代人工"——辅助而非替代是当前的平衡表述。

尾声 · 演进总结

三条贯穿十三年的线

① 相关性定义的三次跃迁

词条命中（集合运算）→ 空间距离（向量检索）→ 生成似然（DSI 的 \(P(j|q)\)、RAG 的答案生成）。每次跃迁都换掉召回层的世界观，但旧世界观都活了下来，成了新世界观的兜底与地基。

② 延迟预算与蒸馏的永恒博弈

最聪明的模型永远太慢：cross-encoder 如此，LLM 更甚。解法始终同构——离线用贵的，在线用蒸的：EBR 的人工标注回流、Gecko 的合成数据、RankGPT 的置换蒸馏、查询改写的 seq2seq 蒸馏。级联漏斗是这场博弈的永久停战线。

③ 评估从成本中心变成引擎

人工标注（贵、慢、稀缺）→ LLM Judge（便宜、快、可复制）后，⑤评估闭环从"验收环节"升级为训练数据的生产车间——Judge 标注 → 蒸馏 → 上线 → 反馈校准 Judge。谁的飞轮转得快，谁的搜索质量迭代就快。

给学习者的一句收尾

读这些论文时别把它们当孤立的"方法"，把每一篇放回骨架里问三个问题：它替换了哪个环节？它靠什么塞进那个环节的延迟预算？它的训练数据从闭环的哪里来？——答完这三问，工业级搜索系统的全貌就在你脑子里了。

先会打分，才谈得上迭代 Grading · AUC / PNR / DCG

相关性分档

人工把 query-doc 相关度标成 4 档（高 / 中高 / 中低 / 低），标注量级达几十万对；训练时映射为标量（如 0, 1/3, 2/3, 1）。这正是正片 Judge 章里 LLM 学着去打的那种标签——先有人工分档体系，才有 UMBRELA 的 0-3 分 prompt。

三层离线指标（pointwise / pairwise / listwise）

AUC：随机取一正一负样本，模型给正样本更高分的概率。
PNR 正逆序比：预测顺序与标注一致的样本对数 ÷ 不一致的对数，越大越好。

$$DCG_p=rel_1+\sum_{i=2}^{p}\frac{rel_i}{\log_2 i}\qquad nDCG_p=\frac{DCG_p}{IDCG_p}$$

位置折损：排得越靠后，相关文档的贡献被 \(\log_2 i\) 打得越狠——对应"用户从上往下看"的行为假设。\(IDCG\)=理想排序下的 DCG，用于跨 query 归一化。（此为课程版写法：首位不折损；标准文献亦常写 \(\log_2(i+1)\)。）

附录 · A4 · 文本匹配分数

正片欠 BM25 的一个公式 TF-IDF · BM25 · Proximity

BM25
$$\mathrm{score}(q,d)=\sum_{t\in q}IDF(t)\cdot\frac{f(t,d)\,(k_1+1)}{f(t,d)+k_1\Big(1-b+b\cdot\frac{|d|}{avgdl}\Big)}$$

\(k_1\)（词频饱和，1.2–2.0）：词频的贡献非线性饱和——一个词出现 3 次和 30 次，相关性差别不大。这是 BM25 对 TF-IDF 的核心洞察：词频与相关性不是线性关系。
\(b\)（长度归一化，0–1）：长文档天然词多占便宜，按 \(|d|/avgdl\) 折算校正。
IDF：全库罕见词权重高（"量子"比"的"值钱）。

词距 Proximity（OkaTP / BM25TP）

"深度"与"学习"在文档里挨得近，比分别散落在无关段落更可能在讨论"深度学习"——词距分数补足词袋模型丢失的顺序信息。

在骨架中的位置变迁

教程明确指出：2020 年前 TF-IDF/BM25/词距是排序阶段的核心人工特征；BERT 兴起后排序普遍弃用文本匹配分数，它退守②召回（文本通道）做候选筛选——与正片"倒排作为混合召回一路"的叙述严丝合缝。

2026 现状：降级不退场

Hybrid 检索的默认 lexical 通道：Elasticsearch（8.9 起）、OpenSearch、Vespa 都把 BM25 + 向量 + RRF（倒数排名融合）作为官方推荐架构；2025 年研究仍显示 BM25 在 BEIR 零样本场景配 reranker 可比肩最佳 dense 模型。演化方向：静态 IDF 权重正被学习型稀疏检索（SPLADE、Elastic ELSER——BERT 给每个词学权重、仍存倒排索引）接管——倒排这个数据结构没死，里面的分数换成了神经网络算的。

最后一米的分数融合，与最快的一条捷径

多目标融合：从规则到模型
$$\mathrm{Score}(q,d)=w_{rel}\,\mathrm{Rel}(q,d)+w_{ctr}\,\mathrm{CTR}(q,d)+w_{qual}\,\mathrm{Qual}(d)+w_{time}\,\mathrm{Fresh}(d)$$

相关性、点击率、质量、时效各算各的分，最后融合出一个排序分。教程的演进建议：初期用人工加权和（可控、可解释、按线上表现调权重）→ 数据充足后换模型融合（LTR：让模型学怎么组合）。单目标必翻车：只优化相关性会推出"相关但过时/低质"的结果。

缓存召回

通用搜索里几百万个高频 query 覆盖每天大部分请求——对它们把 QP 结果、召回结果直接缓存，命中即跳过整条计算链。查询分布的头部集中性，是搜索系统最便宜的加速器。

2026 现状 · LTR/LambdaMART

Airbnb 2025 论文开篇原话：pairwise LTR 是"当今搜索排序技术栈的基石（bedrock）"，其改进版仍在线上部署；RecSys 2025 的 8 周生产 A/B 显示简单 DNN 在点击与营收上开始反超强 GBDT 基线——树模型 vs 神经网络的格局在松动但远未定论。LLM 的角色在上游造语义特征，尚无取代底层 LTR 框架的工业案例。

2026 现状 · 缓存的新形态

从"精确字符串缓存"进化为 semantic cache：在 embedding 空间匹配相似 query，复用检索结果甚至生成好的答案（RAG 场景可削减 30–70% 的 LLM 调用，据 2025 年系统论文与工程实践；大厂官方案例仍少，证据中等）。生成式答案越贵，缓存越值钱——老概念在 LLM 时代反而升值。

2026 现状：产品形态换壳，目标函数没变

经典联想仍在（行业统计约贡献 12–15% 点击，二手数据仅供参考），而 AI 搜索把同一逻辑搬进了对话：Google AI Mode 在每次回答末尾生成可点击的 follow-up 追问，Perplexity 的 related questions 同理——从"预测你想搜什么"变成"替你想好下一个问题"。

本质连线：目标函数始终是"降低表达门槛 + 延长会话"；变化在于生成方式——从 Trie 树查高频日志，变成 LLM 基于当前答案上下文现场生成候选问题。查询词推荐由此从"日志的回声"升级为"理解的延伸"，也顺手接管了正片 fan-out 讲过的"复杂问题分解"能力的交互入口。
