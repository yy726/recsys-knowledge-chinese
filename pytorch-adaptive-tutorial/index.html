1 · 张量 Tensor

PyTorch 的基本数据结构 —— 一个可以在 GPU 上运行、并且能记住自己"来历"的多维数组。

对照:如果你熟悉 NumPy,可以把 torch.Tensor 理解为 np.ndarray 的超集:API 高度相似,但多了两项关键能力 —— ① 可以搬到 GPU 上计算;② 可以参与自动求导(autograd)。

创建张量

import torch
import numpy as np

a = torch.tensor([[1, 2], [3, 4]])      # 从 Python 列表创建,dtype 推断为 int64
b = torch.tensor([1.0, 2.0])            # 含小数 → float32(注意:不是 float64!)
z = torch.zeros(2, 3)                  # 2×3 全 0
o = torch.ones_like(a)                 # 形状同 a 的全 1
r = torch.randn(3, 4)                  # 标准正态分布随机数
n = torch.from_numpy(np.array([1, 2])) # 与 NumPy 数组【共享内存】,改一个另一个也变
易错点:NumPy 默认浮点是 float64,而 PyTorch 默认是 float32(深度学习几乎总用 float32)。用 torch.from_numpy 转换来的张量会保留 float64,常需 .float() 转一下,否则和模型权重的 dtype 不匹配会报错。

形状与常用属性

x = torch.randn(2, 3, 4)
x.shape        # torch.Size([2, 3, 4]),等价于 x.size()
x.dtype        # torch.float32
x.device       # cpu 或 cuda:0 / mps
x.ndim         # 3(维度数)
x.numel()      # 24(元素总数)

s = torch.tensor(3.14)
s.item()       # 3.14 —— 把"单元素张量"取出为 Python 数值(常用于取 loss 值)

变形:view / reshape / squeeze / unsqueeze

x = torch.arange(12)        # [0, 1, ..., 11]
x.view(3, 4)               # 变形为 3×4(不复制数据,要求内存连续)
x.view(-1, 6)              # -1 表示"自动推断"→ 2×6
x.reshape(3, 4)            # 同 view,但遇到不连续内存会自动复制,更稳妥

y = torch.randn(3, 1, 4)
y.squeeze(1).shape          # [3, 4] —— 去掉大小为 1 的维度
y.unsqueeze(0).shape        # [1, 3, 1, 4] —— 插入一个新维度(常用于加 batch 维)
view vs reshape:view 只在张量内存连续(contiguous)时可用,否则抛错;reshape 在必要时会悄悄复制数据。经过 transpose/permute 之后的张量往往不连续,此时要么用 reshape,要么先 .contiguous() 再 view。

运算与广播

a = torch.randn(3, 1)
b = torch.randn(1, 4)
(a + b).shape          # [3, 4] —— 广播规则与 NumPy 完全相同

x = torch.randn(2, 3)
w = torch.randn(3, 4)
x * x                  # 逐元素乘(要求形状可广播)
x @ w                  # 矩阵乘,等价 torch.matmul(x, w) → [2, 4]

x.add_(1)              # 带下划线后缀 = 原地(in-place)操作,直接修改 x 本身

设备:CPU 与 GPU

device = "cuda" if torch.cuda.is_available() else "cpu"
x = torch.randn(2, 3).to(device)   # 把张量搬到 GPU(Mac 上可用 "mps")
model.to(device)                    # 模型也要搬;两者必须在同一设备上,否则报错
对照 TensorFlow:TF 会自动把运算放到可见 GPU 上,而 PyTorch 要求你显式调用 .to(device) 移动张量和模型。这是 PyTorch"显式优于隐式"哲学的体现,初学时最常见的报错之一就是"两个张量不在同一设备"。

2 · Autograd 自动求导

PyTorch 的核心魔法:记录你对张量做过的运算,然后自动算出梯度。

训练神经网络 = 用梯度下降调整参数。手工推导每个参数的梯度不现实,autograd 会在你做前向计算时动态记录一张计算图,调用 backward() 时沿图反向传播,把梯度写进每个参数的 .grad 里。

x = torch.tensor(3.0, requires_grad=True)  # 声明:请追踪对 x 的运算
y = x ** 2 + 4 * x                          # y = x² + 4x(建图)
y.backward()                                # 反向传播,计算 dy/dx
print(x.grad)                               # dy/dx = 2x + 4 = 10.0
对照 TensorFlow:TF 1.x 是"先定义静态图、再喂数据";TF 2.x 用 GradientTape 显式录制。PyTorch 则是动态图(define-by-run):图在每次前向时即时构建,用完即弃,所以可以随意写 Python 的 if/for,调试像普通代码一样直观。

三个关键事实

  • 梯度是累加的。再调一次 backward(),新梯度会加到 .grad 上,而不是覆盖。这就是训练循环里每步都要 optimizer.zero_grad() 的原因。
  • backward 默认只能对标量调用。loss 是标量所以没问题;若对向量调用,需传入 gradient 参数指定权重。
  • 计算图用完即销毁。同一张图上第二次 backward() 会报错,除非第一次传了 retain_graph=True(很少需要)。

不需要梯度的时候

# 推理 / 评估阶段:不建图,省内存、加速
with torch.no_grad():
    pred = model(x_test)

# 从图中"摘下"某个张量:返回一个不带梯度追踪的新张量(共享数据)
detached = y.detach()

# 取 loss 数值用于打印(item() 已隐式脱离计算图)
loss_value = loss.item()
区分:torch.no_grad() 是一个上下文,让其中所有运算都不建图;detach() 是对单个张量操作,把它从已有的图中分离出来。评估模型用前者;想把中间结果当常数继续参与计算用后者。

叶子张量与 .grad

只有叶子张量(用户直接创建、requires_grad=True 的张量,比如模型参数)在 backward 后默认保留 .grad;中间结果的梯度算完即丢,若想查看需先调用 .retain_grad()。

3 · nn.Module 搭建网络

所有神经网络组件的基类:管理参数、组织层次、定义前向计算。

在 PyTorch 里定义模型的标准姿势是:继承 nn.Module,在 __init__ 里声明含参数的层,在 forward 里写数据如何流过这些层。

import torch.nn as nn
import torch.nn.functional as F

class MLP(nn.Module):
    def __init__(self):
        super().__init__()               # 必须先调用父类构造
        self.fc1 = nn.Linear(784, 128)   # 输入 784 维 → 输出 128 维
        self.fc2 = nn.Linear(128, 10)
        self.dropout = nn.Dropout(0.2)

    def forward(self, x):                # 只需定义前向;反向由 autograd 自动完成
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        return self.fc2(x)               # 输出 logits,不加 softmax(见第 4 章)

model = MLP()
out = model(x)   # 调用模型 = 调用 forward(不要直接写 model.forward(x))
为什么层要放在 __init__ 里?把 nn.Linear 等赋值为 self.xxx 时,nn.Module 会自动注册它的参数。这样 model.parameters() 才能把所有权重交给优化器,model.to(device) 才能把它们一起搬到 GPU。在 forward 里临时 new 一个层,参数就不会被注册、不会被训练。

nn.Linear 的形状约定

layer = nn.Linear(in_features=10, out_features=5)
layer.weight.shape    # [5, 10] —— 注意是 (out, in),计算为 y = x Wᵀ + b
layer.bias.shape      # [5]

x = torch.randn(32, 10)   # batch 维在前:32 个样本
layer(x).shape           # [32, 5]

快捷写法:nn.Sequential

model = nn.Sequential(
    nn.Linear(784, 128),
    nn.ReLU(),
    nn.Linear(128, 10),
)   # 数据按顺序依次流过,适合简单"一条道走到黑"的结构

train / eval 模式

model.train()   # 训练模式:Dropout 随机丢弃、BatchNorm 用当前 batch 统计量
model.eval()    # 评估模式:Dropout 关闭、BatchNorm 用训练期累积的滑动均值/方差
易错点:model.eval() 只切换层的行为,并不会关闭梯度计算;评估时应同时使用 with torch.no_grad():。二者作用不同、常常搭配出现。

参数与保存

for p in model.parameters(): ...        # 可训练参数的迭代器(交给优化器用)
model.state_dict()                       # {参数名: 张量} 的有序字典(还含 buffer)
torch.save(model.state_dict(), "m.pt")  # 推荐:只存权重,不存类定义
model.load_state_dict(torch.load("m.pt"))

4 · 损失函数与优化器

损失衡量"错得多离谱",优化器决定"参数往哪儿调"。

常用损失函数

任务损失函数模型输出标签格式
回归nn.MSELoss()连续值连续值,形状同输出
多分类nn.CrossEntropyLoss()原始 logits(N, C)类别下标(N,),不是 one-hot
二分类nn.BCEWithLogitsLoss()原始 logits0/1 浮点
最高频的坑:CrossEntropyLoss 内部已经包含了 log_softmax。如果你在模型末尾自己加了 softmax,相当于做了两次,不会直接报错,但梯度变小、数值不稳、训练明显变差。模型输出裸 logits 就好。同理它的 target 是类别下标(如 tensor([2, 0, 1])),不是 one-hot 向量。

优化器

import torch.optim as optim

optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.9)
optimizer = optim.Adam(model.parameters(), lr=1e-3)   # 最常用的默认选择

优化器在构造时拿到 model.parameters() 的引用,之后每次 step() 就按各参数的 .grad 更新它们。

铁三角:zero_grad → backward → step

optimizer.zero_grad()   # ① 清空上一步残留的梯度(否则会累加!)
loss.backward()          # ② 反向传播,计算所有参数的 .grad
optimizer.step()         # ③ 按梯度更新参数
忘记 zero_grad 会怎样?梯度会一步步累加,相当于方向越来越"陈旧"且幅度失控,典型症状是 loss 剧烈震荡或不收敛。(反过来,故意隔几步才清零 + loss / k,就是"梯度累积"技巧,用小显存模拟大 batch。)
对照 TensorFlow/Keras:model.compile(optimizer, loss) + fit() 把这三步藏了起来;PyTorch 让你显式写出来。代价是多打三行字,好处是任何环节都可以插手 —— 梯度裁剪、累积、自定义更新规则都一目了然。

5 · 完整训练循环

把前四章拼起来:数据 → 前向 → 损失 → 反向 → 更新。

Dataset 与 DataLoader

from torch.utils.data import TensorDataset, DataLoader

dataset = TensorDataset(X, y)                 # 任何实现 __len__ / __getitem__ 的对象都行
loader = DataLoader(dataset, batch_size=64,
                    shuffle=True)             # 训练集要 shuffle,打破样本顺序相关性

标准训练循环(背下来这个骨架)

device = "cuda" if torch.cuda.is_available() else "cpu"
model = MLP().to(device)
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(10):
    model.train()                            # 切训练模式
    for xb, yb in loader:
        xb, yb = xb.to(device), yb.to(device)
        logits = model(xb)                   # ① 前向
        loss = criterion(logits, yb)         # ② 损失
        optimizer.zero_grad()                # ③ 清梯度
        loss.backward()                      # ④ 反向
        optimizer.step()                     # ⑤ 更新

    model.eval()                             # 每个 epoch 结束后评估
    correct = 0
    with torch.no_grad():                    # 评估不建计算图
        for xb, yb in val_loader:
            xb, yb = xb.to(device), yb.to(device)
            pred = model(xb).argmax(dim=1)   # logits 最大的类即预测
            correct += (pred == yb).sum().item()
    print(f"epoch {epoch}: loss={loss.item():.4f}, acc={correct/len(val_set):.2%}")

自查清单(新手 bug 排行榜)

  1. 忘了 optimizer.zero_grad() → 梯度累加,loss 震荡。
  2. 忘了 model.train() / model.eval() 切换 → Dropout/BatchNorm 行为错误,评估分数异常。
  3. 模型和数据不在同一设备 → Expected all tensors to be on the same device。
  4. 给 CrossEntropyLoss 喂了 softmax 后的概率或 one-hot 标签 → 训练慢、效果差。
  5. 用 loss(张量)而不是 loss.item() 做累计统计 → 计算图被一直引用,内存越吃越多。
